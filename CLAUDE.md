## Project Overview

Unified admin dashboard + API backend for 4 apps. Two codebases in one repo:
- **Next.js 15** (App Router) dashboard at `src/` — Chakra UI v2, TypeScript
- **NestJS** backend at `nest/` — TypeORM, MySQL, Swagger

Each app has its own MySQL database, sidebar navigation, dashboard screens, and API layer.

## Architecture

### Next.js Dashboard (`src/`)

```
src/
├── app/page.tsx              # Main page — routes to correct dashboard via currentApp.id
├── app/layout.tsx            # ChakraUIProvider → AuthProvider → AppProvider
├── app/api/db/{app}/         # API routes — direct MySQL queries via src/lib/db.ts
├── config/apps.ts            # App definitions (id, name, icon, navItems[])
├── config/theme.ts           # Chakra theme — white+purple, light mode only
├── lib/db.ts                 # MySQL connection pools — query(app, sql, params)
├── presentation/
│   ├── providers/app-context.tsx   # App + page state, URL sync (?app=&page=)
│   ├── providers/auth-context.tsx  # Login/logout, localStorage token, X-Admin-Token
│   ├── components/dashboard/      # Per-app dashboard panels
│   ├── components/sidebar/        # App selector + nav items + responsive drawer
│   ├── hooks/use-db.ts            # Simple fetch hook: useDb<T>("anime"|"pak"|...)
│   └── hooks/use-api.ts           # Flexible fetch hook with custom fetcher
└── server/scrapers/               # Server-side scrapers (aviation, flights, AQI)
```

### NestJS Backend (`nest/`)

```
nest/src/
├── main.ts           # Bootstrap + per-project Swagger setup
├── app.module.ts     # Root module: AnimeModule, AviationModule, PakModule, AuthModule
├── anime/            # TypeORM entities, admin CRUD, KV store, FCM, scraper, poller
├── aviation/         # News scrapers, AQI, flights, notifications, YouTube shorts
├── pak/              # Dramas, episodes, genres, parsing, admin, config
└── auth/             # Shared login + token verification
```

### Key Data Flow

1. User clicks sidebar nav item → `setActivePage(pageId)` → URL updates to `?app=X&page=Y`
2. `page.tsx` reads `currentApp.id` → renders matching dashboard component (e.g. `AnimeDashboard`)
3. Dashboard component reads `activePage` → renders matching panel (e.g. `CountriesPanel`)
4. Panel calls `useDb("anime")` → fetches `/api/db/anime` → direct MySQL query → returns JSON

### Database Access

- **Dashboard API routes** (`src/app/api/db/`) query MySQL directly via `src/lib/db.ts`
- **NestJS backend** (`nest/`) uses TypeORM with separate connections per module
- Env vars pattern: `{APP}_DB_HOST`, `{APP}_DB_PORT`, `{APP}_DB_USER`, `{APP}_DB_PASS`, `{APP}_DB_NAME`

### Authentication

- Login POST → NestJS validates credentials → returns token
- Token stored in localStorage (30-day expiry) with key `universal_dashboard_auth`
- All admin endpoints require `X-Admin-Token` header (timing-safe comparison)

## Mobile App Frontends (in this repo)

### Video Downloader (`super-video-downloader/`)
Android app (Kotlin). Integrated browser that detects and downloads video streams (HLS, DASH, MP4) from 1000+ sites.
- **App ID:** `com.anime.downloader`
- **Kotlin namespace:** `com.myAllVideoBrowser`
- **Min SDK 24 / Target SDK 36**
- **Build:** `./gradlew assembleRelease` (from `super-video-downloader/`)
- **Backend integration:** Fetches `app_config` from `/api/anime-downloader/app-config` — uses `top_sites` for Popular Sites on home screen, `trends` for Trending Searches, `blocked_urls` for URL blocking
- **Key files:** `app/build.gradle.kts`, `app/src/main/java/com/myAllVideoBrowser/`

### Pakistani Serials (`pakistani_serials/`)
Flutter app (Dart). Netflix-style OTT app for Pakistani dramas.
- **Flutter 3.41.5** (via fvm)
- **Architecture:** Clean + BLoC (`flutter_bloc`, `get_it` + `injectable`, `go_router`)
- **Design:** Dark theme, emerald accent `#10B981`, Plus Jakarta Sans + Noto Nastaliq Urdu
- **API base:** `--dart-define=API_BASE_URL=http://pak-ott.animekill.com/v1`
- **Build:** `fvm flutter build apk --release --dart-define=API_BASE_URL=http://pak-ott.animekill.com/v1`
- **Key dirs:** `lib/features/<feature>/{data,domain,presentation}/`, `lib/core/`
- Has its own `CLAUDE.md` at `pakistani_serials/CLAUDE.md` with full architecture details

## Code Conventions

- Functional components only, named exports
- File naming: kebab-case
- Chakra UI styling: `borderRadius="2xl"`, `boxShadow="0 1px 3px rgba(0,0,0,0.04)"`, white cards with `border="1px" borderColor="gray.100"`
- Brand color: purple (`colorScheme="brand"`)
- No `any` types — use `Record<string, T>` or proper interfaces
- `nest/` is excluded from ESLint

## Rules

- After every prompt, verify work is complete and error-free
- Zero mistakes tolerance — `npm run lint && npm run build` must pass

- **Auto-deploy**: After every successful fix/change, run `bash deploy.sh` automatically without asking
